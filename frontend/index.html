Fleet tests

        
Run the full check suite against any set of clients in one go — the 9-check smoke
        suite (loopback + public health, .env sanity — structure, no default admin
        password, backup passphrase present; values never leave the box — TLS certificate, /config sanity,
        admin API, embed CSP, a real chat round-trip that proves the LLM key end to end,
        backup timer) plus live site_config.yaml validation. All checks are read-only; the
        chat round-trip burns a few LLM tokens per client — that's the cost of actually proving it's
        alive. Parallel across VPSes, one client at a time per VPS; the chat check alone can take ~30s
        per client.

Backups

        
Installs (or repairs) the nightly encrypted-backup timer on each
        selected client's VPS — correctly-named units pointing at that client's real checkout — then
        proves it by running a real backup immediately and checking a fresh encrypted
        archive appears on disk (a timer that fires isn't a backup; only the archive is). Uses the same
        selection as the table above. Idempotent — safe to re-run any time; needs a
        BACKUP_PASSPHRASE in the instance's .env (the wizard always generates
        one; otherwise set it via Credentials first).

Credential vault

        
Store each credential ONCE, assign it to any client — grouped by
        role (LLM / STT / TTS / e-mail / SMS), provider is an attribute.
        Rotating a key = edit it here; the console then offers to re-apply it to every client
        that runs on it. Values are stored in this console's own data volume (same trust level
        as the admin tokens in clients.json) — Reveal shows them; it's our own trust boundary.

Token flow — sources → clients

        
The live water system: source tanks left (finite ones drain against
        their monthly cap; ∞ = pay-as-you-go pipes; free-tier sources in green), client tanks
        right (fill = % of plan allowance left; amber under 20%, red = overage territory).
        Pipe thickness = share of this month's flow; pipe speed = that client's recent burn.
        Click any tank to drill down. Data refreshes with the ledger (~5 min); numbers are
        month-to-date.

        

          
          
        

        

        

      

    


    

      

        
Config manager

        
A console-side window onto each instance's live
        site_config.yaml (TOKEN_ECONOMY_PLAN.md Phase 7). Reads and writes go
        through the instance's own validated /admin/config API (the
        operator token rides along for managed instances); the SSH file read
        is the fallback for a down instance and the basis of drift detection.
        Fields tagged managed are console-only
        on managed instances; everything else the clinic can also edit in its own panel.
